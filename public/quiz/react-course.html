Setup Lesson 01

Installing Express & scaffolding your project

Every React app needs a home. We'll create a Vite-powered React frontend and an Express backend that will eventually serve our Wordle word list via an API.

Step 1 — Scaffold the project

Open your terminal and run the following commands. Vite is a modern build tool — much faster than Create React App. We'll create a wordle-app folder with separate client and server directories.

$ mkdir wordle-app && cd wordle-app
$ npm create vite@latest client -- --template react
✔ Select a framework: React
✔ Select a variant: JavaScript
$ cd client && npm install
$ cd .. && mkdir server && cd server
$ npm init -y
Step 2 — Install Express

Express is a minimal Node.js web framework. We'll use it to create a simple API that returns a random 5-letter word for Wordle.

# Make sure you're inside the /server folder
$ npm install express cors
added 67 packages in 1.2s
✅
Why cors? Your React frontend runs on port 5173 and your Express server on port 3001. Browsers block cross-origin requests by default — the cors package allows them.
Step 3 — Create the Express server

Create a new file server/index.js. This sets up a simple API with one endpoint: GET /api/word which returns a random 5-letter word.

server/index.js
JavaScript 
const express = require('express');
const cors    = require('cors');

const app  = express();
const PORT = 3001;

// Allow requests from our React frontend
app.use(cors({ origin: 'http://localhost:5173' }));
app.use(express.json());

// Word bank — in a real app, load this from a file or database
const WORDS = [
  'react', 'tiger', 'flame', 'storm', 'pixel',
  'crane', 'brave', 'swift', 'plant', 'ghost',
];

// GET /api/word  → returns a random word
app.get('/api/word', (req, res) => {
  const word = WORDS[Math.floor(Math.random() * WORDS.length)];
  res.json({ word });
});

app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
});
Step 4 — Start both servers
# Terminal 1 — start the Express API
$ cd server && node index.js
Server running on http://localhost:3001
 
# Terminal 2 — start the React frontend
$ cd client && npm run dev
VITE v5.0.0 ready in 300 ms
➜ Local: http://localhost:5173/
🚀
Visit http://localhost:3001/api/word in your browser. You should see {"word":"flame"} (or another random word). Your backend is alive!
Structure Lesson 02

Organizing your React folders

Good folder structure is the difference between a codebase that's a joy to work in and one that collapses under its own weight. Here's a battle-tested pattern used by real teams.

The recommended structure

Inside the client/src directory, organize your code into these key folders. Each folder has a single, clear responsibility — this is called separation of concerns.

📁 wordle-app/
📁 client/ ← React frontend
📁 src/
📁 components/ ← Reusable UI pieces (Button, Tile, Grid)
📁 pages/ ← One file per route (HomePage, GamePage)
📁 hooks/ ← Custom React hooks (useWordle, useKeyboard)
📁 utils/ ← Pure helper functions (checkGuess, etc.)
📁 context/ ← React Context providers (GameContext)
📁 assets/ ← Images, fonts, SVGs
📄 App.jsx ← Root component, router lives here
📄 main.jsx ← Entry point, mounts App into DOM
📁 server/ ← Express backend
📄 index.js ← Express app (from Lesson 1)
Why these folders?
1

components/ — reusable UI building blocks

Each component is a small, focused UI element. A Tile.jsx renders a single letter box. A Grid.jsx renders 6 rows of 5 tiles. Components know nothing about game logic — they just display what they're given via props.

2

pages/ — one file per URL route

Pages are the top-level views that React Router renders. HomePage.jsx is what you see at /. GamePage.jsx is at /game. Pages import components and hooks to build full screens.

3

hooks/ — extract & reuse stateful logic

Custom hooks like useWordle.js contain all the game logic (checking guesses, tracking state). By moving logic out of components, you make both easier to test and understand.

4

utils/ — pure functions with zero side effects

A function like checkGuess(guess, answer) that takes inputs and returns outputs lives in utils/. Pure functions are trivially easy to unit test.

Creating the files
$ cd client/src
$ mkdir components pages hooks utils context assets
$ touch pages/HomePage.jsx pages/GamePage.jsx
$ touch hooks/useWordle.js utils/checkGuess.js
$ touch components/Tile.jsx components/Grid.jsx components/Keyboard.jsx
⚠️
Keep it flat when possible. Don't create nested subfolders unless you have 10+ files in one folder. Over-nesting is a common trap. Start flat and refactor when needed.
Routing Lesson 03

Adding React Router

React is a single-page application framework — there's only one HTML file. React Router lets you simulate multiple pages by swapping components based on the URL, without a full page reload.

Install React Router
$ cd client && npm install react-router-dom
Set up routes in App.jsx

Replace the contents of App.jsx with the router setup. We wrap everything in <BrowserRouter> and define our routes inside <Routes>.

client/src/App.jsx
JSX 
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import HomePage from './pages/HomePage';
import GamePage from './pages/GamePage';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/"      element={<HomePage />} />
        <Route path="/game"  element={<GamePage />} />
        <Route path="*"      element={<h1>404 – Not Found</h1>} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;
Create the pages

Key Router concepts
🔀

<BrowserRouter>

Wraps your whole app. Provides routing context to all child components. Only one per app.

🗺️

<Routes> + <Route>

Defines which component to render for each URL path. path="*" is the catch-all 404 route.

🔗

<Link>

Replaces <a href> tags. Navigates without a page reload, keeping React state intact.

🧭

useNavigate()

A hook for programmatic navigation. Use inside event handlers: navigate('/game').

Wordle Lesson 04

Wordle: Game Logic

Now the fun part. We'll build the core Wordle logic: fetching the secret word from our Express API, checking guesses, and tracking game state with React hooks.

The checkGuess utility

First, let's write the pure function that compares a guess to the answer and returns a status for each letter: "correct", "present", or "absent".

client/src/utils/checkGuess.js
JavaScript 
/**
 * Compares a guess against the answer.
 * Returns an array of 5 objects: { letter, status }
 * status: 'correct' | 'present' | 'absent'
 */
export function checkGuess(guess, answer) {
  const result  = Array(5).fill(null);
  const ansArr  = answer.split('');
  const used    = Array(5).fill(false);

  // First pass: find correct (green) letters
  for (let i = 0; i < 5; i++) {
    if (guess[i] === ansArr[i]) {
      result[i] = { letter: guess[i], status: 'correct' };
      used[i]   = true;
    }
  }

  // Second pass: find present (yellow) and absent (grey)
  for (let i = 0; i < 5; i++) {
    if (result[i]) continue; // already correct

    const idx = ansArr.findIndex(
      (l, j) => l === guess[i] && !used[j]
    );

    if (idx !== -1) {
      result[i]  = { letter: guess[i], status: 'present' };
      used[idx] = true;
    } else {
      result[i] = { letter: guess[i], status: 'absent' };
    }
  }

  return result;
}
The useWordle custom hook

This hook is the heart of the game. It fetches the secret word from our Express API, manages the guess history, handles keyboard input, and determines win/loss state.

client/src/hooks/useWordle.js
JavaScript 
import { useState, useEffect, useCallback } from 'react';
import { checkGuess } from '../utils/checkGuess';

export function useWordle() {
  const [answer,  setAnswer]  = useState('');
  const [guesses, setGuesses] = useState([]); // array of result arrays
  const [current, setCurrent] = useState(''); // letters typed so far
  const [gameOver,setGameOver] = useState(false);
  const [won,     setWon]     = useState(false);

  // Fetch the secret word from our Express API on mount
  useEffect(() => {
    fetch('http://localhost:3001/api/word')
      .then(r => r.json())
      .then(data => setAnswer(data.word));
  }, []);

  const submitGuess = useCallback(() => {
    if (current.length !== 5) return; // must be 5 letters

    const result   = checkGuess(current, answer);
    const newGuesses = [...guesses, result];

    setGuesses(newGuesses);
    setCurrent('');

    if (current === answer) {
      setWon(true); setGameOver(true);
    } else if (newGuesses.length === 6) {
      setGameOver(true);
    }
  }, [current, answer, guesses]);

  // Keyboard listener
  useEffect(() => {
    const onKey = (e) => {
      if (gameOver) return;
      if (e.key === 'Enter')     { submitGuess(); return; }
      if (e.key === 'Backspace')  { setCurrent(c => c.slice(0, -1)); return; }
      if (e.key.match(/^[a-z]$/) && current.length < 5) {
        setCurrent(c => c + e.key);
      }
    };
    window.addEventListener('keydown', onKey);
    return () => window.removeEventListener('keydown', onKey);
  }, [submitGuess, gameOver, current]);

  return { answer, guesses, current, gameOver, won };
}
🧠
Why a custom hook? We could put all this logic directly in GamePage.jsx, but by extracting it into useWordle, the component stays clean and readable — and we can reuse the logic anywhere.
Wordle Lesson 05

Wordle: UI Components

The logic is done. Now let's build the visual layer — the Tile, Grid, and Keyboard components — and wire them all together in GamePage.

Tile component

The Tile is the smallest unit: a single letter square with a color based on its status.

client/src/components/Tile.jsx
JSX 
const COLORS = {
  correct: '#6aaa64',
  present: '#c9b458',
  absent:  '#787c7e',
};

export default function Tile({ letter = '', status }) {
  const bg = status ? COLORS[status] : 'transparent';

  return (
    <div style={{
      width: '60px', height: '60px',
      border: `2px solid ${status ? bg : '#d3d6da'}`,
      background: bg,
      display: 'flex', alignItems: 'center', justifyContent: 'center',
      fontSize: '2rem', fontWeight: 'bold',
      color: status ? '#fff' : '#000',
      textTransform: 'uppercase',
      transition: 'background 0.3s, border-color 0.3s',
    }}>
      {letter}
    </div>
  );
}
Grid component
client/src/components/Grid.jsx
JSX 
import Tile from './Tile';

export default function Grid({ guesses, current }) {
  // Build 6 rows: completed guesses + current row + empty rows
  const rows = [...guesses];

  // Add the current (in-progress) row
  if (rows.length < 6) {
    const currentRow = current.split('').map(l => ({ letter: l, status: null }));
    while (currentRow.length < 5) currentRow.push({ letter: '', status: null });
    rows.push(currentRow);
  }

  // Fill remaining rows with empty tiles
  while (rows.length < 6) {
    rows.push(Array(5).fill({ letter: '', status: null }));
  }

  return (
    <div style={{ display: 'flex', flexDirection: 'column', gap: '5px' }}>
      {rows.map((row, ri) => (
        <div key={ri} style={{ display: 'flex', gap: '5px' }}>
          {row.map((tile, ti) => (
            <Tile key={ti} letter={tile.letter} status={tile.status} />
          ))}
        </div>
      ))}
    </div>
  );
}
Putting it together in GamePage
client/src/pages/GamePage.jsx
JSX 
import { Link } from 'react-router-dom';
import { useWordle } from '../hooks/useWordle';
import Grid from '../components/Grid';

export default function GamePage() {
  const { guesses, current, gameOver, won, answer } = useWordle();

  return (
    <div style={{ textAlign: 'center', padding: '20px' }}>
      <Link to="/">← Home</Link>
      <h1>Wordle</h1>

      <Grid guesses={guesses} current={current} />

      {gameOver && (
        <div style={{ marginTop: '20px' }}>
          {won
            ? <h2>🎉 You got it!</h2>
            : <h2>The word was: {answer.toUpperCase()}</h2>
          }
          <button onClick={() => window.location.reload()}>
            Play Again
          </button>
        </div>
      )}
    </div>
  );
}
